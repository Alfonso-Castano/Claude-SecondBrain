## What to do

Per `CONTEXT.md`'s Implementation Decisions ("Markdown tooling ... `markdownlint-cli2` + a link-checker as real dev dependencies") and this feature's `RESEARCH.md` (§1), adopt `markdownlint-cli2` and `markdown-link-check` — both real npm packages, so the entire tooling surface lives inside `package.json` with no separate binary/toolchain install. Do **not** substitute `lychee` — it's not an npm package and would require a second install mechanism outside `package.json` (see `RESEARCH.md` for the full reasoning).

1. Create `package.json`:
```json
{
  "name": "claude-secondbrain",
  "private": true,
  "version": "1.0.0",
  "description": "Tooling for linting and link-checking the Second Brain wiki.",
  "scripts": {
    "lint:md": "markdownlint-cli2",
    "lint:links": "find . -name \"*.md\" -not -path \"./node_modules/*\" -not -path \"./raw/*\" -print0 | xargs -0 -n1 markdown-link-check -c .markdown-link-check.json",
    "lint": "npm run lint:md && npm run lint:links"
  },
  "devDependencies": {}
}
```
Note: `lint:links` relies on `find`/`xargs` (available via Git Bash on this machine, per the project environment). This is a reasonable starting invocation, not a polished pipeline — running an actual lint pass is explicitly out of scope for this feature (nothing exists yet to lint); this script will get exercised for real at the first future lint-pass feature.

2. Create `.markdownlint-cli2.jsonc`:
```jsonc
{
  // Base ruleset, with two rules relaxed for prose wiki pages.
  "config": {
    "default": true,
    "MD013": false, // line length — wiki pages are prose, not code; don't force hard wraps
    "MD033": false  // allow inline HTML if a page ever needs it
  },
  "globs": ["**/*.md"],
  "ignores": ["node_modules/**", "raw/**"]
}
```
`raw/` is excluded because it holds unprocessed source material (never edited, not written to this schema) — linting it against wiki markdown style would be a category error.

3. Create `.markdown-link-check.json`:
```json
{
  "retryOn429": true,
  "retryCount": 3,
  "timeout": "10s",
  "aliveStatusCodes": [200, 206, 301, 302, 429]
}
```

4. Create `.gitignore`:
```
node_modules/
```

5. Run `npm install --save-dev markdownlint-cli2 markdown-link-check` from the repo root. This populates `devDependencies` in `package.json` at their current resolved versions and generates `package-lock.json` — do not hand-write version numbers into `package.json` yourself.
